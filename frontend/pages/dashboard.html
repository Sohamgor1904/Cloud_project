🧠 Business Logic Member 4
Priority Formula
priority = (fill% × 0.1)
+ urgency_noise(0–2)

Converts fill % to 0–10 scale with urgency factor. Range 0–12. Source: backend/services/priorityService.js

Route Threshold
bins.filter(b => b.priority >= 7.0)
.slice(0, 10)

Bins ≥ 7.0 qualify. Top 10 highest-priority enter the route. Source: backend/services/routeService.js

Greedy Ordering
score = 0.6 × normFill
− 0.4 × normDist

At each step picks the bin maximising this score. Real Haversine distances. Source: frontend/js/route.js

Alert Thresholds
≥ 75% → WARNING
≥ 90% → OVERFLOW_RISK

Alert shape: { bin_id, location, fill_pct, alert }. Source: backend/services/alertService.js

🔄 Data Flow — Simulate → Map → Dispatch
  1. 1User clicks Run Simulation — POST /api/bins/simulate sent from frontend/js/dashboard.js
  2. 2backend/controllers/binController.js calls simulationService.js → generates random fill levels for all 22 bins
  3. 3New fill values written to fill_history table in PostgreSQL (or kept in memory). DB write is mandatory — not in-memory!
  4. 4alertService.js fires alerts for bins ≥ 75%. Response: { bins, alerts } returned to frontend
  5. 5frontend/js/route.js greedyOptimizeRoute() runs in the browser — orders candidates by score
  6. 6frontend/js/map.js redraws markers + route polyline. charts.js updates all charts. Alerts shown in nav bar.

📋 Non-Negotiable Rules
✅ DO
  • Keep frontend, backend, and database in separate folders
  • Each page's CSS and JS in its own file
  • User data lives only in data/users.json
  • Test each API endpoint before wiring the UI
  • Commit at end of every work session
❌ DON'T
  • Mix business logic into HTML files
  • Hardcode credentials in source code
  • Change API response shapes without team sync
  • Commit .env or node_modules/
  • Save debugging for the morning of the demo
