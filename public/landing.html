Real-Time Simulation

Click Simulate — IoT sensor data is generated, priority scores computed, and the city map updates instantly. No page reload needed.

🧠

Priority Scoring

Each bin scores: priority = (fill% × 0.1) + noise(0–2). Bins ≥ 7.0 enter the route. Top 10 are dispatched.

🚛

Route Optimiser

Greedy nearest-neighbour algorithm: 60% fill urgency + 40% travel proximity using real Haversine distances. Prevents backtracking.

📊

Analytics Dashboard

Chart.js doughnut (fill distribution), horizontal bar (top-10 priorities), and zone-average bar charts. All update on each simulation.

🔮

Overflow Predictions

AI-modelled fill rates predict overflow time per bin — 2h, 4h, and 8h projections shown in the Analytics tab. Alert engine fires at 75% and 90%.

System Flow

From sensor to dispatch in seconds

1

Simulate IoT Sensors

Click Run Simulation. The system generates realistic fill percentages for all 22 bins across GIFT City — mimicking real IoT sensor data.

→
2

Calculate Priority

Each bin gets a priority score: fill% × 0.1 + urgency(0–2). The score scales from 0–12. Bins ≥ 7.0 qualify for today's route.

→
3

Persist to PostgreSQL

New fill values are written to fill_history in real time. The routing endpoint reads from the DB — not memory — so data never goes stale.

→
4

Optimise the Route

Greedy algorithm selects the next bin by maximising 0.6 × fill urgency − 0.4 × distance. Truck starts at GIFT City depot.

→
5

Live Map Dispatch

The city map updates with numbered route stops, a dashed route polyline, and overflow alerts. Driver follows this exact order — no wasted stops.
