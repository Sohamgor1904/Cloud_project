🧠 Business Logic (Member 4)
Priority Formula
priority = (fill_pct × 0.1)
+ urgency_noise(0–2)

Converts fill % to 0–10 scale and adds urgency factor. Range: 0–12.

Route Threshold
bins.filter(b => b.priority >= 7.0)
.top10

Bins scoring ≥ 7.0 qualify. Top 10 highest-priority enter the route.

Route Generation
score = 0.6 × normFill
− 0.4 × normDist

Greedy nearest-neighbour: 60% fill urgency + 40% travel proximity. Real Haversine distances.

Alert Threshold
≥ 75% → WARNING
≥ 90% → OVERFLOW_RISK

Alert shape: { bin_id, location, fill_pct, alert }

🔄 Data Flow — Simulate → Map
  1. 1User clicks Run Simulation → POST /api/bins/simulate via fetch()
  2. 2Express backend calls simulateBins() → generates random fill levels for all 22 bins
  3. 3New fill values written to fill_history table in PostgreSQL. DB write is mandatory — not in-memory!
  4. 4GET /api/bins reads latest_bin_fills VIEW → calculatePriority() applied per bin
  5. 5GET /api/route/today → generateRoute() filters ≥ 7.0 → greedyOptimizeRoute() orders stops
  6. 6Browser re-renders: map markers update, route polyline redraws, charts refresh, alert banners fire

Q: "How is the priority calculated?"
"Priority = fill percentage scaled to 0–10, plus a small urgency factor 0–2. Bins scoring above 7 enter today's route, ordered by a greedy algorithm that balances waste urgency (60%) with travel distance (40%)."
